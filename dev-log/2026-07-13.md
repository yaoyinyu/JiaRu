- 任务开始前复核 `docs/technical-whitepaper.md` v1.1.10 的功能状态总表、v4–v7 结论、验证门和已知限制；本轮只补桌面端内存证据，不把桌面结果冒充移动真机验收。
- 新增 `scripts/benchmark-recognition-memory.ps1`，通过 Playwright CLI 在真实 `/ar-tryon` 页面连续执行上传、Worker/WebGPU 推理和关闭弹窗，同时采集 JS heap、Chromium 全进程 working set/private bytes、模型版本与执行时间。
- 新增 `scripts/verify-recognition-memory.ts` 及正反单元测试；门禁要求至少20次、JS首末窗口增长不超过32MiB、浏览器私有内存增长不超过128MiB，且不能几乎全程单调增长。
- v6 完成20次真实识别：JS heap 峰值19.86MiB、首末窗口增长1.69MiB；Chromium private memory 峰值929.50MiB、首末窗口增长121.81MiB；working set 峰值840.63MiB。JS/私有内存最长连续增长为14/7次，序列均出现回落，桌面暂定稳定性门通过。
- 全浏览器进程内存包含 renderer、GPU、utility、缓存和自动化进程，不能视为模型独占内存；Android、iPhone、iPad 的128–150MiB移动增量目标继续保留为真机输入项。
- 白皮书同步为 v1.1.11，新增桌面内存结论与变更记录；实施进度 `M3-T3-DEVICE` 从 USER INPUT 更新为 PARTIAL，并新增 `docs/recognition-memory-gate.md` 和机器可读摘要。
