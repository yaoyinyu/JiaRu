# 开发日志 - 2026-07-02

## 今日概览

今天主要推进了两条线：

1. Phase 4：围绕美甲纹理提取质量，补稳定化、质量提示、透明裁切验收和诊断展示。
2. Phase 5：围绕训练发布治理链路，补齐 handoff、trace、history 和自动治理主链。

## Phase 4：纹理提取与前端可解释性

### 1. 候选方向稳定化

涉及：

- `src/lib/nail-texture-recognition/postprocess.ts`
- `src/lib/nail-texture-recognition/index.ts`
- `tests/nail-texture-preprocess-postprocess.test.ts`
- `docs/nail-texture-angle-stabilization.md`

完成内容：

- 新增候选角度稳定逻辑，避免接近圆形或主轴不稳定的候选出现 angle 抖动。
- 当候选自身方向不可靠时，优先借用同组内可靠候选的共享方向。
- 当整组都不可靠时，统一回退到默认竖直方向。

结果：

- 纹理裁切和展示更稳定。
- 为后续透明裁切、UI 质检提示打下基础。

### 2. NailArtPicker 质量提示

涉及：

- `src/components/nail-art-picker-quality.ts`
- `src/components/NailArtPicker.tsx`
- `tests/nail-art-picker-quality.test.ts`
- `docs/nail-art-picker-quality-hints.md`

完成内容：

- 新增 UI 侧质量提示映射层。
- 将内部 warning code 转成可读提示文案。
- 将低置信度、存在 warning、提取质量不达标等情况统一收口为“建议复核”。

结果：

- 用户能知道为什么某个候选需要复核。
- 产品层面开始具备基础可解释性。

### 3. 透明裁切 / 边缘羽化专项验收

涉及：

- `tests/nail-texture-mask-pipeline.test.ts`
- `docs/transparent-mask-texture-verification.md`

- 在测试中验证透明边缘、羽化过渡、高光修复是否真实反映到最终纹理输出。

结果：

- 纹理贴图产物质量第一次被纳入明确的自动化验收。

### 4. extraction diagnostics UI 展示

## Phase 5：训练发布治理链路

### 5. 安全发布入口

涉及：

- `scripts/promote-approved-release.ts`
- `tests/promote-approved-release.test.ts`
- `docs/promote-approved-release.md`

完成内容：

- 新增“先审再发”的安全发布入口。
- 只有 decision report 允许发布时，才继续注册 candidate。
- 对 `manual_review` 增加显式人工放行开关。
- 输出 `promotion-report.json` 记录 decision、manifest、register summary 和阻断原因。

### 6. release trace index

涉及：

- `scripts/build-release-trace-index.ts`

完成内容：

- 形成 `release-trace-index.json`，可回答某个 candidate 对应的数据来源、审计结果、决策状态和注册结果。

### 7. release history manifest
