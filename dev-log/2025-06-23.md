## 技术决策记录

### 决策 1: ArView paintNails 提取为纯函数
**问题**: 原 paintNails 在 useEffect 内部定义，依赖闭包中的变量，ESLint 报 "used before defined"。
**方案**: 移到组件顶层作为普通函数，参数显式传入 (ctx, lm, colors, w, h)。
**理由**: 纯函数无副作用依赖，更易测试和维护；消除闭包陷阱。

### 决策 2: colRef pattern 用于颜色同步
**问题**: nailColors props 变化时，onResults 回调中的 colRef 需要同步更新。
**方案**: 独立 useEffect 监听 nailColors 变化，更新 colRef.current。
**理由**: 避免 onResults 闭包捕获旧值；ref 更新不触发重渲染。

### 决策 3: NailCanvas 向后兼容接口设计
**问题**: 编辑器需要五指选色，但不应破坏原有单色用法。
**方案**: selectedColor 和 nailColors 都设为可选 (?), 内部优先级 nailColors > selectedColor > 默认值。
**理由**: 渐进式迁移，其他调用方无需改动。

### 决策 4: SVG 使用 HTML entity 编码中文
**问题**: 原 SVG 直接写中文字符串，在某些环境下显示为乱码（娓愬彉）。
**方案**: 所有中文使用 &#xNNNN; HTML entity 编码（如 渐变裸粉 = &#28176;&#21464;&#35074;&#31881;）。
**理由**: 彻底消除编码依赖，SVG 作为 XML 在任何环境下都能正确解析。

### 决策 5: 诊断面板条件渲染
**问题**: 原 ArView 诊断面板永久显示，影响用户体验。
**方案**: `{status !== "ready" && <诊断面板/>}` + `{status !== "ready" && <遮罩/>}`。
**理由**: ready 状态下用户不需要看到调试信息；错误/加载状态保留诊断便于排查。
