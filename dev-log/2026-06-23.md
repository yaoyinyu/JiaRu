# 2026-06-23 Dev Log

## AR 手心/手背朝向检测实现

### 目标
手心朝镜头时不渲染美甲纹理（手心无指甲），仅手背朝镜头时渲染。

### 技术方案审核
- 原方案的叉积法向量法 **被弃用**（依赖 handedness，左右手符号相反）
- 改用**深度差法**（palmZ - knuckleZ），不依赖 handedness
- 辅助方案改为 **4 指投票**（排除拇指，拇指 z 不稳定）

### 实现步骤
1. ✅ 添加常量：`DEPTH_DIFF_THRESHOLD`、`FINGER_Z_VOTE_THRESHOLD`、`OUT_OF_FRAME_THRESHOLD`、`VOTE_FINGERS`
2. ✅ 添加 `palmDepthSmoothRef`（EMA 平滑深度差）和 `orientation` state（UI 显示）
3. ✅ 实现 `shouldRenderNails()` 函数：
   - 边界检查（手掌关键点 x/y 超出 [0.1, 0.9] → 降级）
   - 方案 B：深度差判断（palmZ vs knuckleZ）
   - 方案 C：4 指投票
   - 融合决策：双方案一致 → high confidence；不一致 → 采用 B 结果 + low confidence

### 关键参数
| 参数 | 值 | 说明 |
|------|-----|------|
| `DEPTH_DIFF_THRESHOLD` | 0.003 | depthDiff > 此值 = 手背 |
| `FINGER_Z_VOTE_THRESHOLD` | 0.002 | 单指投票阈值 |
| `OUT_OF_FRAME_THRESHOLD` | 0.1 | x/y 超出 [0.1, 0.9] = 出画面 |
| `EMA_ALPHA_PALM` | 0.3 | 深度差 EMA 平滑因子 |

### 已知限制
- z 坐标是单目视觉的相对深度估计，精度低于 x/y
- 侧手时 `|depthDiff|` 很小，安全降级不渲染
- EMA 平滑有 ~100-167ms 延迟（3-5 帧 @30fps），快速翻手时有短暂过渡
- 阈值 0.003 需实测调优
