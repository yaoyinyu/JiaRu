# AR 朝向检测改进方案 — 根因分析与三层修复

**时间**: 2026-06-24 01:55
**问题**: 当前深度差法误判率高，美甲贴图仍被贴到手心侧

## 根因分析

### 为什么深度差法不够准确？

1. **z 坐标噪声大**：MediaPipe z 是单目视觉的相对深度估计，精度远低于 x/y。手部在画面中占比小、光照变化、手指弯曲时 z 值波动剧烈
2. **阈值难调**：`0.003` 是猜测值，实际 z 值分布范围因摄像头/距离/手型而异，没有通用最优值
3. **EMA 平滑延迟**：α=0.3 有 100-167ms 延迟，快速翻手时过渡态会误渲染
4. **4 指投票与深度差用同样的 z 数据**：两个方案不独立，z 不准时不一致概率低，但一致时也可能都错

### 核心矛盾

MediaPipe Hands **21 个关键点的 z 坐标精度不足以可靠区分手心/手背**。这是模型能力的根本限制，不是算法问题。

## 改进方案：三层渐进式修复

### Layer 1（立即可做）：启用 `multiHandedness` + 叉积法

**关键发现**：MediaPipe JS 版**确实返回** `multiHandedness` 字段，只是当前 TS 接口没声明。

有了 handedness（Left/Right 标签），叉积法向量方案就能用——根据左右手选择不同的叉积顺序，使 `n.z` 符号统一。

**优点**：
- 叉积法用 x/y 坐标（精度高），不依赖 z
- handedness 是模型分类结果，比 z 深度可靠
- 实现简单：只需扩展接口 + 加 handedness 判断

**风险**：
- 自拍模式视频镜像翻转后 handedness 标签会反转（Left↔Right）
- 需要根据镜像状态补偿

### Layer 2（Layer 1 不够时）：拇指位置辅助验证

拇指在左右手的位置不同：
- 右手手背朝镜头：拇指在左侧
- 右手手心朝镜头：拇指在右侧
- 左手相反

用拇指 TIP(4) 相对手掌中心(0,5,9,17)的 x 位置 + handedness 验证朝向。

**优点**：用 x 坐标（精度最高），完全不依赖 z
**缺点**：拇指弯曲或并指时失效

### Layer 3（Layer 1+2 都不够时）：换更先进的技术方案

| 方案 | 精度 | 复杂度 | 适合场景 |
|------|:----:|:------:|----------|
| **MediaPipe HandLandmarker (新版 API)** | ⭐⭐⭐ | 中 | 替代旧 hands.js，支持 `worldLandmarks`（真实 3D 坐标） |
| **TensorFlow.js HandPose** | ⭐⭐ | 中 | 社区维护，精度类似 MediaPipe |
| **Apple Vision Framework** | ⭐⭐⭐⭐ | 高 | 仅 iOS，精度最高 |
| **手部分割模型（Hand Segmentation）** | ⭐⭐⭐⭐ | 高 | U-Net 轻量分割，直接区分手心/手背像素 |
| **自训练轻量分类器** | ⭐⭐⭐ | 中 | 收集 100-500 张手心/手背图片，训练 MobileNetV3 微调 |

**推荐**：先试 Layer 1+2，如果仍不够，换 **MediaPipe HandLandmarker 新版 API**（`worldLandmarks` 提供真实 3D 坐标，z 精度更高）。

## 实施计划

### 立即实施 Layer 1 + Layer 2

1. 扩展 `HandResults` 接口，声明 `multiHandedness`
2. 在 `onResults` 中读取 handedness 标签
3. 实现叉积法向量 + handedness 补偿
4. 实现拇指位置辅助验证
5. 三方案融合：叉积(x/y) + 拇指位置(x) + 深度差(z) 三重投票
6. 阈值调优：从保守开始（宁可漏渲染也不要误渲染）
