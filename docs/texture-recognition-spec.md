# JiaRu 美甲纹理识别方案 — 技术需求文档

> 本文档是给 AI 模型（如 GPT/Claude）的输入，要求其输出一套完整的、可落地的美甲纹理自动识别技术方案。

---

## 1. 背景与目标

### 1.1 当前方案的问题
JiaRu 项目（美甲 AR 试戴应用）目前使用 **TextureCropper.tsx** 组件让用户手动拖拽选框裁剪指甲纹理。这种方式存在以下痛点：

- **用户体验差**：用户需要手动框选指甲区域，操作繁琐，容易选偏
- **缺乏智能化**：没有自动识别指甲位置、方向、大小的能力
- **纹理质量不稳定**：手动选区可能选到手背皮肤、背景或其他非指甲区域
- **无法批量处理**：多指图片需要用户逐个框选

### 1.2 目标
设计一套 **美甲纹理自动识别系统**，能够：
1. 接收用户上传的手部照片（任意角度、任意肤色、任意光照）
2. 自动检测 5 根手指的指甲区域位置、大小、旋转角度
3. 从每根手指的指甲区域提取高质量纹理贴图（ImageBitmap）
4. 输出可直接用于 AR 试戴渲染的纹理资源

---

## 2. 技术约束

### 2.1 运行环境
- **纯浏览器端运行**，不允许发送到服务器处理
- 运行在 Web Worker 或 OffscreenCanvas 中，不阻塞主线程 UI 响应
- 支持 Chrome 120+ / Safari 17+ / Firefox 120+（现代浏览器）
- 移动端性能要求：中端手机（骁龙 865 / A13）上处理耗时 ≤ 3 秒

### 2.2 与现有系统的集成约束
- 输出格式必须兼容现有 `texture.ts` 中的 `ImageBitmap` 接口
- 识别结果数据结构必须兼容 `NailGeometry`（来自 `nail-geometry.ts`）：
  ```typescript
  interface NailGeometry {
    cx: number;       // 指甲中心 X 坐标（原图像素）
    cy: number;       // 指甲中心 Y 坐标（原图像素）
    length: number;   // 指甲长度（像素）
    width: number;    // 指甲宽度（像素）
    angle: number;    // 指甲旋转角度（弧度）
  }
  ```
- 最大纹理尺寸不超过 512×512（当前为 256，可扩展）
- 必须复用 `extractTexture()` 进行最终的纹理裁剪和缩放

### 2.3 输入图片约束
- 格式：JPEG / PNG / WebP
- 分辨率：最小 320×320，最大 4096×4096
- 内容：单张手部照片，包含 1~2 只手，手指部分可见即可
- 用户可能上传非手部图片（风景照、宠物照等），需要优雅降级

## 3. 方案设计要求

AI 需要输出一份完整的技术方案，方案必须涵盖以下所有方面：

### 3.1 核心技术路线选择

需要对比并选择以下一种或多种技术路线，说明理由：

**路线 A：MediaPipe Hands + 几何推断**
- 复用项目已有的 MediaPipe Hands 手部关键点检测
- 基于 21 个 3D 关键点，用几何关系推算每指指甲区域
- 优点：项目已有基础设施，代码复用率高
- 缺点：关键点精度有限（归一化坐标），指甲区域推断可能有偏差

**路线 B：专用指甲分割模型（Segment Anything / SAM 2）**
- 使用 Meta 的 Segment Anything Model 或其移动端版本 MobileSAM
- 训练/微调一个指甲分割模型
- 优点：分割精度最高
- 缺点：模型体积大（数百 MB），浏览器端推理困难

**路线 C：YOLO / EfficientNet 检测 + 裁剪**
- 训练一个指甲区域检测模型（Bounding Box 回归）
- 优点：速度快，模型小
- 缺点：需要标注数据集

**路线 D：混合方案（推荐考虑）**
- MediaPipe 检测手部 → 粗定位指甲区域 → 精细算法/轻量模型优化
- 结合各路线优势

**要求**：必须分析每种路线在当前项目约束下的可行性，给出明确推荐。

### 3.2 指甲区域检测算法

无论选择哪条路线，方案必须详细说明：

1. **输入预处理**：
   - 图片缩放策略（保持宽高比？最大边限制？）
   - 色彩空间转换（RGB → Lab / HSV / YCrCb？是否需要？）
   - 降噪滤波（高斯模糊？双边滤波？）

2. **手部检测与定位**：
   - 如何确定图片中手的数量和位置
   - 如何处理多只手（检测所有？还是只取最大手？）
   - 如何处理手部不完整/部分出画的情况

3. **指甲区域精确定位**：
   - 如何从手部关键点或特征推断每指指甲的具体位置
   - 指甲的长宽比、角度如何估计
   - 如何处理不同手指（拇指与其他四指差异大）
   - 如何处理美甲已经存在的图片（用户可能已做了美甲）

4. **可见性过滤**：
   - 如何判断某指指甲是否可见（朝向镜头？被遮挡？）
   - 参考现有 ArView.tsx 中的 `shouldRenderNails()` 逻辑

### 3.3 纹理提取与后处理

1. **裁剪策略**：
   - 如何根据检测到的指甲区域生成裁剪框
   - 裁剪框的 padding 策略（是否需要包含部分指根/指尖？）
   - 如何处理裁剪框超出图片边界的情况

2. **纹理质量优化**：
   - 是否需要颜色校正（白平衡？）
   - 是否需要去除指甲表面反光/高光
   - 是否需要边缘羽化处理

3. **资源管理**：
   - ImageBitmap 的生命周期管理
   - 多纹理并发处理的内存控制
   - 错误处理和回退机制

### 3.4 性能优化方案

1. **计算优化**：
   - 哪些步骤可以并行（Web Worker / OffscreenCanvas）
   - 哪些步骤可以跳过（如图片过大时先降采样）
   - MediaPipe 模型加载优化（懒加载？缓存？）

2. **内存优化**：
   - 大图处理的内存峰值控制
   - ImageBitmap 的及时释放
   - 中间结果的复用

3. **用户体验优化**：
   - 检测过程中的 loading 状态反馈
   - 失败时的友好提示
   - 检测结果的可视化预览（让用户确认/微调）

### 3.5 鲁棒性与边界情况

方案必须覆盖以下场景的处理策略：

| 场景 | 要求 |
|------|------|
| 非手部图片 | 优雅提示用户，不 crash |
| 光线极暗/极亮 | 尽可能检测，置信度低时提示用户手动调整 |
| 手部距离镜头极近/极远 | 自适应缩放策略 |
| 手指弯曲/手势特殊（比耶、握拳） | 弯折手指的指甲不可见，应跳过 |
| 佩戴手套/戒指 | 尽可能检测可见指甲 |
| 多只手重叠 | 优先检测前景手 |
| 图片分辨率极高（>4K） | 降采样到合理尺寸再处理 |
| 图片分辨率极低（<100px） | 直接拒绝，提示重新上传 |
| 透明背景 PNG | 正确处理 alpha 通道 |
| 方形/圆形/异形美甲 | 纹理提取应覆盖整个指甲区域 |

### 3.6 与现有 NailArtPicker 的整合

方案需要说明：

1. **是否替换** TextureCropper（纯手动裁剪）？
2. **还是增强** NailArtPicker（在自动检测基础上保留手动微调）？
3. 自动检测结果如何与现有的 9 手柄交互系统结合
4. 置信度低的检测区域是否自动回退到手动模式

## 4. 输出要求

AI 需要输出以下内容：

### 4.1 技术方案文档（Markdown）
包含：
- 推荐的技術路线及详细理由
- 完整的算法流程图（用 Mermaid 或文字描述）
- 每个步骤的伪代码/关键算法描述
- 参数选择依据（阈值、尺寸、缩放因子等）
- 性能预估（各步骤耗时估算）

### 4.2 代码结构规划
包含：
- 新的模块/文件应该如何组织
- 每个文件的责任边界
- 与现有文件的依赖关系图
- 公共接口的 TypeScript 类型定义

### 4.3 实施路线图
包含：
- Phase 1：基础检测（MediaPipe + 几何推断）
- Phase 2：质量优化（颜色校正、边缘处理）
- Phase 3：鲁棒性提升（边界情况处理）
- Phase 4：性能优化（Web Worker、并行处理）
- 每个 phase 的验收标准和优先级

### 4.4 风险评估
包含：
- 技术风险点（哪些步骤可能达不到预期效果）
- 备选方案（如果主推路线失败，有什么 Plan B）
- 已知局限性（哪些场景无法处理）

## 5. 参考信息

### 5.1 项目现有相关代码
- `src/lib/texture.ts` — 纹理提取和缩放（必须兼容）
- `src/lib/nail-geometry.ts` — 指甲几何计算接口（必须兼容）
- `src/components/TextureCropper.tsx` — 手动裁剪器（可能被替换）
- `src/components/NailArtPicker.tsx` — 多纹理拣选器（需要整合）
- `src/components/ArView.tsx` — AR 渲染引擎（纹理消费者）
- `src/lib/nail-image-detection.ts` — 静态图片指甲检测算法（可参考）

### 5.2 外部依赖
- MediaPipe Hands 0.4（已在项目中通过 CDN 引入）
- 可选：WebNN API（Chrome 130+ 神经网络加速）
- 可选：ONNX Runtime Web（跨浏览器模型推理）
- 可选：Transformers.js（Hugging Face 浏览器端推理）

### 5.3 关键参数
- `MAX_TEXTURE_SIZE`: 当前 256，建议可扩展至 512
- `MAX_CANVAS_DIM`: 当前 1024（TextureCropper）/ 800（NailArtPicker）
- 手指名称映射：`["拇指", "食指", "中指", "无名指", "小指"]`
- 指甲几何参数：每指独立的 `NAIL_OFFSET_RATIOS`、`NAIL_LENGTH_RATIOS`、`NAIL_WIDTH_RATIOS`
