# 参考图美甲纹理提取修复方案

## 验收样本

- 文件：`C:\Users\YaoYinyu\.codex\attachments\e3e3f943-acb1-45f4-899a-a3492814fd2a\image-1.jpg`
- 尺寸：860 × 573
- 图像特征：只拍到四根手指的末端和中段，手掌、手腕以及大部分骨骼关键点均不在画面中。

## 已确认根因

当前自动选区完全依赖 MediaPipe Hands 的 21 点整手骨架。参考图属于常见的美甲特写裁切图，缺少完整手掌轮廓，因此 MediaPipe 返回空结果。现有代码随后直接提示“未在图片中检测到手指”，没有针对美甲特写图的降级检测路径。

这不是简单调低 MediaPipe 置信度可以解决的问题。继续降低阈值只会增加错误骨架，无法可靠覆盖缺少手掌的照片。

## 修复策略

1. 保留 MediaPipe 作为完整手部照片的首选检测器。
2. 新增不依赖手部骨架的本地图像分析降级路径：
   - 估计皮肤色分布；
   - 计算“非皮肤/高色差/高饱和度”候选掩码；
   - 进行形态学闭运算，连接美甲内部被金线和高光切开的区域；
   - 提取连通区域；
   - 根据面积、长宽、位置、皮肤邻接比例和内部颜色变化评分；
   - 使用区域二阶矩估计中心、长宽和旋转角。
3. 对候选区域去重并按横向顺序分配到可见手指。四指特写默认映射为食指、中指、无名指、小指；五个候选时映射全部五指。
4. 自动检测失败时保留手动选区，但错误提示必须区分“整手骨架未检测到”和“图像降级检测也未找到候选”。
5. 提取阶段继续使用原图像素，而不是显示缩略图，确保纹理清晰度。

## 分步验证

1. 为纯图像检测函数增加单元测试，覆盖：
   - 参考图至少检出 4 个候选；
   - 候选中心落在四个真实甲片附近；
   - 候选尺寸和旋转角有效；
   - 纯肤色/空白图不会产生大量误报。
2. 运行 ESLint、单元测试和生产构建。
3. 在浏览器中上传同一参考图，验证：
   - 不再出现“未在图片中检测到手指”；
   - 画面出现四个自动选区；
   - 点击“完成分配”后生成四份 `ImageBitmap` 纹理；
   - 返回 AR 页面后纹理模式可用。

## 完成标准

只有当参考图在真实界面中无需手动添加选区即可检测并成功提取纹理，同时测试和构建全部通过，才视为完成。
