# 指甲贴合与参考图自动识别 V2 技术方案

**创建时间**：2026-06-29  
**状态**：实施中  
**涉及文件**：`src/lib/nail-geometry.ts`、`src/components/ArView.tsx`、`src/components/NailArtPicker.tsx`

## 1. 问题与根因

### 1.1 AR 贴合

现有实现从 MediaPipe 的 TIP/DIP 推导指甲中心、长度、宽度与方向，但存在以下结构性问题：

1. 指甲路径的长轴位于 Canvas 局部坐标的 Y 轴，指尖位于 `-Y`；代码却直接使用
   `atan2(TIP-DIP)` 作为旋转角，导致路径长轴与手指方向相差约 90°。
2. 指甲宽度只由远端指骨长度乘固定比例得到，没有结合当前手指投影宽度和透视状态。
3. 中心、长度、宽度各自计算且只平滑关键点，尺寸仍会随关键点噪声呼吸抖动。
4. AR 与参考图识别复制了两套几何公式，参数和修复容易漂移。

### 1.2 参考图自动识别

MediaPipe Hands 只检测 21 个手部骨骼关键点，并不直接输出指甲轮廓。现有实现还有两个确定性坐标错误：

1. 调用处已把归一化关键点转换成 Canvas 像素，`computeNailRegions` 内又乘一次 Canvas
   宽高，形成二次缩放，选区通常落到画布外。
2. 选区保存在缩放后 Canvas 坐标中，提取纹理时却直接用于原图，造成裁剪位置和尺寸错位。

因此 V2 采用“骨骼先验定位 + 图像证据精修 + 可编辑回退”，而不是把手部关键点误称为
“指甲检测”。

## 2. V2 设计

### 2.1 单一共享几何模型

新增纯函数模块 `src/lib/nail-geometry.ts`，统一计算：

- 方向：`TIP - DIP`；退化时回退 `DIP - PIP`。
- 旋转：`atan2(dy, dx) + PI / 2`，使局部 `-Y` 指向指尖。
- 中心：从 TIP 沿手指方向向根部回退逐指比例。
- 长宽：以 TIP-DIP 投影长度为基础，加入透视压缩保护并限制合理范围。
- 坐标：输入、输出必须显式声明为归一化坐标或目标画布像素，禁止隐式二次缩放。

AR 和参考图必须调用同一函数，避免公式漂移。

### 2.2 AR 时序稳定

每根手指新增几何状态 EMA，对 `cx/cy/length/width/rotation` 一起平滑：

- 位置与尺寸使用自适应 EMA；移动快时提高响应，静止时提高稳定性。
- 角度使用最短圆周差平滑，避免跨越 `-PI/PI` 时跳变。
- 检测突然跳跃时限制单帧尺寸变化，防止贴图“呼吸”。
- 纹理和纯色均先裁剪到同一个贝塞尔指甲路径，禁止矩形纹理溢出甲面。

### 2.3 参考图图像精修

骨骼模型给出五个候选区域后，对每个候选区域执行本地像素分析：

1. 在候选中心附近搜索小范围平移、宽度和长度缩放组合。
2. 比较候选内部与周围皮肤环的颜色差、饱和度差和边缘响应。
3. 选择得分最高且仍位于合理解剖范围内的候选。
4. 图像证据不足时保留骨骼先验，并在 UI 中标记为“建议检查”，不静默失败。

该过程完全在浏览器本地进行，不上传图片。

### 2.4 正确纹理提取

- 将 Canvas 选区通过 `1 / displayScale` 映射回原图。
- 以指甲中心为原点，按正确旋转角反向变换原图。
- 输出画布固定为“宽 × 长”，透明区域保留，贝塞尔路径外像素不进入纹理。
- AR 绘制纹理前再次使用目标手指路径裁剪。

## 3. 验收标准

### 3.1 自动化与代码验收

| 编号 | 标准 |
|---|---|
| C1 | `eslint` 0 errors |
| C2 | Next.js 生产构建成功 |
| C3 | AR 与参考图只使用共享几何函数，不保留重复公式 |
| C4 | 水平、垂直、斜向三组关键点中，指甲局部 `-Y` 与 DIP→TIP 方向误差小于 0.5° |
| C5 | 归一化点映射到 800×600 画布后，所有候选中心均在画布范围内，不发生二次缩放 |
| C6 | 选区从显示画布映射回原图后，中心和长宽误差不超过 1 原图像素 |
| C7 | 纹理提取结果方向为宽×长，路径外像素透明 |
| C8 | 无手、CDN 失败、低图像证据均能进入可编辑手动模式 |

### 3.2 参考图功能验收

使用至少 10 张图片：5 张单色/跳色清晰手背图、3 张复杂图案手背图、2 张无手图片。

| 编号 | 标准 |
|---|---|
| D1 | 8 张含手图片中至少 7 张检测到手 |
| D2 | 检测到手的图片中，每张至少生成 4 个画布内候选选区 |
| D3 | 清晰正视手背图中，候选中心落在真实甲面内的比例 ≥ 80% |
| D4 | 五指标签顺序与 MediaPipe 手指标号一致率 100% |
| D5 | 无手图片不生成伪选区，并显示手动添加入口 |
| D6 | 用户调整后的选区能够正确提取并分配到目标手指 |

### 3.3 AR 真机验收

至少覆盖一台 Android Chrome 和一台 iPhone Safari；每台执行正视、左右倾斜约 30°、
靠近、远离、缓慢移动五组动作。

| 编号 | 标准 |
|---|---|
| A1 | 正视手背时至少 4/5 个指甲中心落在真实甲面内，连续保持 5 秒 |
| A2 | 指甲长轴与真实手指远端方向夹角中位数 ≤ 10°，不得再出现约 90°横置 |
| A3 | 静止 5 秒时，中心抖动标准差 ≤ 对应指甲宽度的 8% |
| A4 | 静止 5 秒时，长宽抖动峰峰值 ≤ 平均尺寸的 12% |
| A5 | 30°缓慢倾斜时，纹理不溢出贝塞尔甲面，且不发生明显横跳 |
| A6 | 手心朝镜头时不得持续渲染；偶发误帧连续不超过 3 帧 |
| A7 | 目标设备推理与渲染帧率 ≥ 15 FPS |

## 4. 验收策略

本轮完成后先执行 C1-C8；失败即继续修复。D/A 类需要真实图片和设备，项目提供清晰的
选区覆盖层和诊断数据供复测。未取得真机证据前，状态只能标记为“代码验收通过、真机待验”，
不得写成“完美贴合”或“全部完成”。
