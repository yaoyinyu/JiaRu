**创建时间**: 2026-06-24
**目标**: 支持任意手势（比耶、握拳、点赞等）下，仅对伸出且指甲可见的手指渲染美甲纹理
**对应文件**: `src/components/ArView.tsx`

## 1. 问题分析

### 1.1 当前方案在非全开手时的局限

当前的三信号融合（变A/B/C）全部基于手指与摄像头的**相对朝向**：

| 信号 | 方法 | 依赖 | 手势场景失效原因 |
|------|------|------|-----------------|
| A | TIP.z - DIP.z | z 深度差 | 弯曲手指的 TIP/DIP z 关系不变 |
| B | TIP.z - PIP.z | z 深度差 | 同上 |
| C | len2D/len3D 缩短比 | 透视投影 | 弯曲后 2D 和 3D 长度都变，比值变化不大 |

**结果**: 用户比耶时，系统仍然尝试渲染所有 5 指的指甲—包括弯折隐藏的无名指和小指。

### 1.2 手势场景列举

| 手势 | 可见指甲 | 应渲染 | 不应渲染 |
|------|---------|:------:|:--------:|
| 🖐️ 五指张开 | 全部 | 五指 | — |
| ✌️ 比耶 | 食指+中指 | 食指,中指 | 拇指,无名指,小指 |
| 🤘 摇滚 | 食指+小指 | 食指,小指 | 拇指,中指,无名指 |
| ☝️ 食指指 | 食指 | 食指 | 其他四指 |
| 👍 点赞 | 拇指 | 拇指 | 其他四指 |
| 👊 握拳 | 无 | — | 五指 |
| 🤙 打电话 | 拇指+小指 | 拇指,小指 | 食指,中指,无名指 |
| ✊ 握拳半露 | 部分可见 | 对应手指 | 其余 |

### 1.3 手指伸展角：物理原理

当手指弯折时，指甲面向下或侧向，不再朝向镜头。检测弯折的关键是**远端指骨（DIP→TIP）相对于中段指骨（PIP→DIP）的夹角**：

```
伸直：PIP→DIP 和 DIP→TIP 在同一直线 → 夹角 ≈ 0°
弯折：DIP→TIP 偏离 PIP→DIP 方向 → 夹角 > 0°
```

```
完全伸直:    PIP─────DIP─────TIP    夹角 ≈ 0-15°
轻度弯曲:    PIP─────DIP            夹角 ≈ 15-35°
                         ↘ TIP
深度弯折:    PIP─────DIP            夹角 > 35°
                           TIP
```

**优势**: 这个角度只用 x/y 坐标（MediaPipe 精度最高），不依赖 z，且物理含义直白。

## 2. 技术方案

### 2.1 新增信号 D：手指伸展角

```typescript
/**
 * 计算手指伸展角（PIP→DIP 与 DIP→TIP 的夹角）
 * 0 rad = 完全伸直，值越大弯折越深
 * 只用 x/y 坐标，精度最高
 */
function computeFingerExtensionAngle(
  pip: { x: number; y: number },
  dip: { x: number; y: number },
  tip: { x: number; y: number }
): number {
  const v1x = dip.x - pip.x;   // 中段指骨方向
  const v1y = dip.y - pip.y;
  const v2x = tip.x - dip.x;   // 远端指骨方向
  const v2y = tip.y - dip.y;

  // 点积 / 叉积 → atan2 → 夹角
  const dot = v1x * v2x + v1y * v2y;
  const cross = v1x * v2y - v1y * v2x;
  return Math.abs(Math.atan2(cross, dot));
}
```

### 2.2 四信号融合

```
信号 A — TIP.z vs DIP.z（z 深度差）
信号 B — TIP.z vs PIP.z（不同关节印证）
信号 C — 透视缩短比（x/y 几何）
信号 D — 手指伸展角 ★新增（x/y 几何，不受 z 噪声影响）
```

```typescript
// 信号 D
const extensionAngle = computeFingerExtensionAngle(s.pip, s.dip, s.tip);
const sigD = extensionAngle < EXTENSION_ANGLE_THRESHOLD;
```

### 2.3 投票策略

```typescript
// 强否定：四信号全部否定才隐藏
if (!sigA && !sigB && !sigC && !sigD) return false;

// 多数投票：≥ 2 信号认可 → 可见（4 信号中 2 个通过）
const votes = [sigA, sigB, sigC, sigD].filter(Boolean).length;
return votes >= 2;
```

### 2.4 各手势下的信号行为

| 手势 | 手指 | A(z差) | B(pip-z) | C(缩短比) | D(伸展角) | 投票 | 结果 |
|------|------|:------:|:---------:|:---------:|:---------:|:----:|:----:|
| 🖐️ 五指 | 伸直五指 | ✓ | ✓ | ✓ | ✓ | 4 | ✅ 渲染 |
| ✌️ 比耶 | 食指 | ✓ | ✓ | ✓ | ✓ | 4 | ✅ 渲染 |
| ✌️ 比耶 | 无名指 | ✓ | ✓ | ✓ | ✗(弯折) | 3 | ❌ 跳过 |
| 👊 握拳 | 任何指 | ✗ | ✗ | ✗ | ✗(弯折) | 0 | ❌ 跳过 |
| 👍 点赞 | 拇指 | ✓ | ✓ | ✓ | ✓ | 4 | ✅ 渲染 |
| 👍 点赞 | 其余 | ✗ | ✗ | ✗ | ✗ | 0 | ❌ 跳过 |

### 2.5 阈值选择

```
EXTENSION_ANGLE_THRESHOLD = 0.6  // 弧度 ≈ 35°
```

| 手指状态 | 典型角度 | sigD |
|----------|:--------:|:----:|
| 完全伸直 | 0-10° | ✓ |
| 自然微曲 | 10-20° | ✓ |
| 轻微弯折（指甲仍部分可见） | 20-35° | ✓（宽松保留） |
| 明显弯折（指甲朝下） | 35-60° | ✗ |
| 紧握拳 | 60-90° | ✗ |

## 5. 风险与回退

| 风险 | 概率 | 缓解 |
|------|:----:|------|
| 手指自然微曲(<20°)被误判弯折 | 低 | 阈值 0.6(35°) 覆盖自然微曲 |
| 从侧面角度观察伸直手指时夹角偏大 | 低 | 投影几何导致侧面角度 ≈ 真实夹角 |
| D 信号覆盖所有其他信号，过度主导 | 中 | 投票制（≥2/4），D 只能贡献 1 票 |
