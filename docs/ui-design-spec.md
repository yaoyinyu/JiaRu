# UI设计规范 — 甲如

## 1. 品牌色

```
主色调（品牌色）：
  - 玫瑰粉（主色）:    #E8A0BF
  - 深玫瑰（强调色）:  #D4749D
  - 极浅粉（背景色）:  #FFF5F7
  - 纯白（背景色）:    #FFFFFF
  - 深灰（文字色）:    #4A4A4A

辅助色：
  - 浅灰（次要文字）:  #9CA3AF
  - 粉色渐变:          from #FFF5F7 to #FFE4EC
```

## 2. 字体

```
中文字体: PingFang SC, Microsoft YaHei, Noto Sans SC, sans-serif
英文字体: Inter, system-ui, sans-serif
字号系统: text-xs(12) / sm(14) / base(16) / lg(18) / xl(20) / 2xl(24) / 3xl(30)
```

## 3. 组件规范

### 按钮
```
圆角: rounded-2xl (16px) 或 rounded-full
最小高度: 48px（触屏友好）
内边距: px-6 py-3
阴影: shadow-sm, hover:shadow-md
动效: transition-all duration-200, active:scale-95
```

### 卡片
```
圆角: rounded-2xl
内边距: p-4 或 p-6
阴影: shadow-sm
背景: bg-white
边框: border border-pink-100
```

### 颜色选择按钮
```
圆形: rounded-full
尺寸: w-10 h-10 (40px, 触屏友好)
选中态: ring-2 ring-offset-2 ring-pink-400
```

### 导航栏
```
高度: h-16 (64px)
背景: bg-white/80 backdrop-blur-md
固定: fixed top-0
```

## 4. 页面布局

### 首页
```
Logo + 品牌名称（顶部居中）
四个功能入口（网格2×2或垂直排列）
底部标语
```

### 编辑器
```
顶部: 返回按钮 + 标题
中间: 照片画布（全宽自适应）
底部: 颜色选择面板 + 操作按钮（撤销/重置/保存）
```

### 移动端适配
```
断点: sm(640px) / md(768px) / lg(1024px)
优先移动端设计，按钮最小48px
触摸目标间距 ≥ 8px
```

## 5. 交互动效

```
按钮点击: scale 0.97 → 1.0
页面切换: fade in (opacity 0→1, 300ms)
颜色选中: ring 缩放动画
保存成功: 短暂显示 ✓ 提示
```

## 6. AR 模块 UI 规范

### AR 页面布局
```
顶部: 返回按钮 + 标题「AR 试戴」
中间: 摄像头画面（全屏，object-cover 裁剪）
      - 未启动时：占位图 + 「开启摄像头」按钮居中
      - 启动后：实时视频流 + Canvas 覆盖层
底部: 手指选择 tab（5指） + 颜色面板 + 模式切换（纯色/纹理）
```

### 摄像头启动交互
```
触发方式: 用户手动点击「开启摄像头」按钮
原因: 移动端浏览器拦截自动 getUserMedia
按钮样式: 大圈按钮，居中，品牌色渐变
加载状态: 按钮变为“加载中…” + 旋转动画
错误状态: 显示具体错误信息（权限拒绝/设备未找到/其他）
```

### 朝向指示器
```
位置: 底部状态栏上方
显示: 🖐️ 手背（绿色）/ ✋ 手心（橙色）/ 🤚 侧手（灰色）
条件: status === "ready" 时显示
与手部计数共存
```

### 手指选择 Tab
```
5 个手指按钮（拇指/食指/中指/无名指/小指）
当前选中: ring-2 ring-pink-400
未选中: bg-white/50
点击: 切换 activeFinger，颜色面板同步显示该手指当前颜色
```

### 纯色/纹理模式切换
```
两个 toggle 按钮
纯色模式: 颜色面板选色 → fillStyle 纯色填充
纹理模式: 上传纹理图片 → 柱面曲率变形绘制
```

### 诊断面板（仅开发用）
```
位置: 右上角
显示条件: status !== "ready"（非 ready 状态）
内容: 摄像头状态、模型加载状态、错误信息
```
