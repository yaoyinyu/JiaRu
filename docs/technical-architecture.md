## 3. 数据流

### MVP 涂色流程（全前端）
```
用户上传照片 → FileReader → Canvas加载
→ 用户选色/涂抹 → Canvas绘制叠加层
→ 保存 → Canvas.toDataURL → 下载
（全程在浏览器内存中，无网络请求）
```

### AI 生成流程（需后端）
```
用户输入描述 → 前端POST /api/generate-ai
→ API Route调用OpenAI → 返回图片URL
→ 前端展示 → 可保存/分享
（仅传文本，不传照片）
```

### AR 实时试戴流程
```
用户点击"开启摄像头" → 请求摄像头权限
→ 启动前置摄像头 video 流
→ 加载 MediaPipe HandLandmarker 模型（约5-10秒）
→ 每帧检测手部21个关键点
→ 根据指尖(PIP)位置计算指甲区域
→ Canvas覆盖层绘制半透明椭圆指甲
→ 跟随手指实时更新
（全程本地处理，不录制不上传）
```

## 5. AR 模块开发规划

### 5.1 当前状态与问题

**状态：** AR 模块基础结构已搭建（ArView组件 + AR页面），但在实际设备上无法正常工作。

**已发现问题：**
1. **摄像头方向**：当前使用 `facingMode: "environment"`（后置摄像头），美甲试戴应使用前置（自拍）摄像头
2. **MediaPipe 加载**：TS类型定义不完整，`@mediapipe/tasks-vision` 的 CDN 路径和模型路径需要验证
3. **错误日志不足**：catch 块未打印具体错误信息，无法定位失败环节
4. **组件生命周期**：`isStarted` 切换时 ArView 重新挂载，但 MediaPipe 每次都要重新加载（5-10秒），体验差
5. **Canvas 渲染性能**：每帧清除重绘可能造成闪烁

### 5.2 分步修复计划

```
Step 1: 添加详细错误日志
   → 在 catch 块 console.error 具体错误
   → 状态显示具体错误信息
   → 验证 MediaPipe 模型是否可加载

Step 2: 修复摄像头调用
   → facingMode 改为 "user"（前置摄像头）
   → 添加摄像头启动超时处理
   → 优化权限拒绝后的用户提示

Step 3: 优化 MediaPipe 加载体验
   → 添加加载进度提示
   → 添加重试机制
   → 组件卸载时正确释放资源

Step 4: 修复手部检测循环
   → 确保 canvas 与 video 同步
   → 优化 frameSkip 逻辑
   → 修复 nailColors 响应式更新

Step 5: 优化指甲贴合效果
   → 改进指甲位置和大小计算
   → 添加光泽/高光效果
   → 支持多只手同时检测

Step 6: 性能优化
   → 降低检测分辨率
   → 优化渲染循环
   → 低端设备降级提示
```

### 5.3 技术要点

#### MediaPipe HandLandmarker 关键点索引
```
0: 手腕
4: 拇指指尖      3: 拇指IP关节
8: 食指指尖      7: 食指PIP关节
12: 中指指尖     11: 中指PIP关节
16: 无名指指尖   15: 无名指PIP关节
20: 小指指尖     19: 小指PIP关节
```

#### 指甲区域计算
```
指尖 → 远侧指关节 方向向量
指甲中心 = 指尖位置沿手指方向偏移 15%
指甲长度 = 指尖到关节距离 × 0.55
指甲宽度 = 指尖到关节距离 × 0.45
指甲旋转 = 手指方向的角度
```

#### Canvas 绘制
```
1. 清除上一帧
2. 遍历每只手、每根手指
3. 计算指甲椭圆参数（中心、长轴、短轴、旋转角）
4. 绘制半透明填充椭圆（颜色）
5. 绘制白色高光椭圆（光泽感）
```
