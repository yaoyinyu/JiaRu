# AR 指甲识别准确率优化 — 参数调优与逻辑改进

**创建时间**: 2026-06-24
**目标**: 提升 AR 手部指甲识别的准确率（减少漏识别、改善贴合度）
**对应文件**: `src/components/ArView.tsx`

---

## 1. 问题诊断

### 1.1 当前准确率低的原因分析

| # | 问题 | 影响 | 证据 |
|---|------|:----:|------|
| 1 | 五指共用同一个指甲中心偏移量 (0.28) | 拇指和尾指指甲位置偏差 | `TIP_OFFSET_RATIO` 为单值 |
| 2 | 指甲大小比与真实解剖有偏差 | 指甲形状不准确，与真指甲轮廓不匹配 | `FINGER_LENGTH/WIDTH_RATIOS` 为估测值 |
| 3 | 可见性判定强否定过于激进 | 手偏转时部分可见指甲被误判为隐藏 | `!sigA && !sigC` 仅需 2 信号否定即隐藏 |
| 4 | 透视缩短比阈值 0.65 偏严 | 45° 偏转时本应可见的指甲被隐藏 | `FORESHORTEN_THRESHOLD = 0.65` |
| 5 | 信号 B 阈值 0.003 偏严 | TIP-PIP z 差不稳定场景漏判 | `NAIL_PALM_Z_THRESHOLD_B = 0.003` |
| 6 | 手指方向向量取 DIP-PIP | 指尖弯曲时方向有偏差 | 方向向量从 PIP→DIP 再到 TIP |

### 1.2 优化原则

1. **减少漏判优先**：宁可偶尔多渲染，也尽量覆盖所有可见指甲
2. **解剖学校准**：所有参数参考真实手指/指甲比例
3. **逐指差异化**：5 指各自的参数独立调优
4. **改动可回退**：只调整常量和阈值，不改变架构

---

## 2. 优化方案

### 2.1 逐指指甲中心偏移（新增 `FINGER_OFFSET_RATIOS`）

**现状**: `TIP_OFFSET_RATIO = 0.28`（所有手指相同）

**问题**: 拇指指甲覆盖指端更多（靠近指尖），尾指指甲也更靠近指尖。统一 0.28 导致拇指和尾指偏移过多。

**改进**: 新增逐指偏移数组

```
FINGER_OFFSET_RATIOS = [0.22, 0.28, 0.28, 0.26, 0.24]
拇指=0.22：更靠近指尖（拇指甲床占指端比例大）
食指=0.28：标准
中指=0.28：标准
无名指=0.26：稍近
尾指=0.24：更靠近指尖（尾指甲床占比大）
```

### 2.2 指甲大小比调优（解剖学校准）

**现状**:
```
FINGER_LENGTH_RATIOS = [0.55, 0.58, 0.60, 0.56, 0.52]
FINGER_WIDTH_RATIOS  = [0.55, 0.50, 0.48, 0.45, 0.40]
```

**问题**: 长度比偏大（0.55-0.60），宽度比对拇指偏宽（0.55）。

**改进**（基于真实甲床比例）:
```
FINGER_LENGTH_RATIOS = [0.50, 0.55, 0.58, 0.54, 0.48]
FINGER_WIDTH_RATIOS  = [0.52, 0.48, 0.46, 0.44, 0.36]
```

| 手指 | 原长度 | 新长度 | 原宽度 | 新宽度 | 说明 |
|------|:------:|:------:|:------:|:------:|------|
| 拇指 | 0.55 | 0.50 | 0.55 | 0.52 | 拇指甲床占远端指骨比例约 50% |
| 食指 | 0.58 | 0.55 | 0.50 | 0.48 | 食指甲床稍收窄 |
| 中指 | 0.60 | 0.58 | 0.48 | 0.46 | 最长的指甲，宽度收窄 |
| 无名指 | 0.56 | 0.54 | 0.45 | 0.44 | 微调 |
| 尾指 | 0.52 | 0.48 | 0.40 | 0.36 | 尾指甲最窄小 |

### 2.3 可见性判定强否定降级

**现状**:
```typescript
if (!sigA && !sigC) return false;  // 2 信号否定即隐藏
```

**问题**: sigA（z 差）和 sigC（缩短比）两个信号否定就判定隐藏，过于激进。混合态场景下，sigA 可能因 z 噪声为 false，但指甲实际可见。

**改进**:
```typescript
if (!sigA && !sigB && !sigC) return false;  // 3 信号全否定才隐藏
```

这样需要所有三个信号一致否定才判定为隐藏，大幅减少漏判。

### 2.4 透视缩短比阈值放宽

**现状**: `FORESHORTEN_THRESHOLD = 0.65`

**物理含义**: 手指与镜头平面夹角 > ~40° 时判定隐藏

**问题**: 40° 偏转时指甲仍然部分可见（尤其从侧面视角）

**改进**: `FORESHORTEN_THRESHOLD = 0.55`

**物理含义**: 手指与镜头平面夹角 > ~55° 时判定隐藏。在 55° 偏转时指甲侧缘仍可见，但甲面大部分朝向侧方，合理隐藏。

### 2.5 信号 B 阈值放宽

**现状**: `NAIL_PALM_Z_THRESHOLD_B = 0.003`

**改进**: `NAIL_PALM_Z_THRESHOLD_B = 0.004`

PIP 比 DIP 更靠指根，z 噪声更大。阈值放宽到 0.004 减少漏判。

### 2.6 方向向量使用 TIP-DIP（更贴近指甲方向）

**现状**: 方向向量 = DIP - PIP（指骨方向）

**问题**: 指甲长在远端指骨（TIP-DIP 段），指甲方向更接近于 TIP-DIP 向量而非 DIP-PIP。当 PIP 或 DIP 有轻微弯曲时，DIP-PIP 方向与 TIP-DIP 方向有偏差。

**改进**: 默认使用 TIP-DIP 作为方向向量（更贴近指甲实际朝向），DIP-PIP 仅在没有 TIP 时作为降级。

```
// 新逻辑（交换优先级）
if (rawTip && rawDip) {
  fx = s.tip.x - s.dip.x; fy = s.tip.y - s.dip.y; // TIP-DIP 优先
} else if (rawPip) {
  fx = s.dip.x - s.pip.x; fy = s.dip.y - s.pip.y; // DIP-PIP 降级
}
```

**影响**: 指甲旋转角度更匹配实际指甲朝向，指甲位置（基于 TIP 偏移）也更准确。

### 2.7 参数变更汇总

| 参数 | 改前 | 改后 | 说明 |
|------|:----:|:----:|------|
| `FINGER_OFFSET_RATIOS` | 无 | [0.22, 0.28, 0.28, 0.26, 0.24] | 新增逐指偏移数组 |
| `FINGER_LENGTH_RATIOS` | [0.55,0.58,0.60,0.56,0.52] | [0.50,0.55,0.58,0.54,0.48] | 长度比整体减小 |
| `FINGER_WIDTH_RATIOS` | [0.55,0.50,0.48,0.45,0.40] | [0.52,0.48,0.46,0.44,0.36] | 宽度比收窄 |
| `FORESHORTEN_THRESHOLD` | 0.65 | 0.55 | 放宽(40°→55°才判定隐藏) |
| `NAIL_PALM_Z_THRESHOLD_B` | 0.003 | 0.004 | 信号 B 放宽 |
| 强否定逻辑 | `!sigA && !sigC` | `!sigA && !sigB && !sigC` | 三信号全否定才隐藏 |
| 方向向量优先级 | DIP-PIP 优先 | TIP-DIP 优先 | 更贴近指甲朝向 |
